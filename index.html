What is Monoriel?

Monoriel is an attempt to create a super simple Web Development Mini Framework, and believe me, monoriel is quiet small. But it can create web apps using a simplistic approach by means of http verbs or something RESTful as it is known.

Why do i must use Monoriel?

Because is simple and can process Slim, Haml and Scss templates to generate Web Pages an Web Apps. You can use any rack middleware with monoriel. You can create before an after filters an helper methods for your app. Every method just need to be prefixed with the http method it responds, like

def index
  render 'index'
end

def post_client
  client = Client.new
  client.name = params["name"]
  client.create
end

def put_client(id)
  client = clients.read id
  client.name = params["name"]
  client.update
end

def delete_client(id)
  clients.destroy id
end

as you see, you don't need to prefix GET methods and index is the default method when the / route is browsed.

Using layouts for templates

As you can note in the code above, there is a method called render. This method is not part of Monoriel, in fact is a Helper method but you can write it:

helpers do
  def render template
    slim :layout do
       slim template.to_sym
    end
  end
  def redirect dir
    @res.redirect dir
  end
  def params
    @r.params
  end
end

and that's all, now you have layout supporting for Slim templates engine, but you can do the same for haml, or erb, just code render method as shown above.I have included the redirect and params methods which are something common for use in an app.
